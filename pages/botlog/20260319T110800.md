### 출발: semantic-memory는 이름이 아니다

agent-config 안에 pi-extension으로 있던 `semantic-memory` 를 별도 리포로 분리한다. 이유: Pi뿐 아니라 Claude Code, OpenClaw 등 여러 에이전트 런타임에서 쓸 수 있어야 하기 때문.

그런데 "semantic-memory"는 **기능 설명** 이지 **이름** 이 아니다. "회사 지식베이스를 RAG 합니다"와 구분이 안 된다. 이 프로젝트는 그런 용도가 아니다.

### 이 프로젝트가 하는 일

인간과 에이전트가 시간 축 위에 날것으로 기록한 **존재의 전체** 에 대한 인터페이스.

현재 담고 있는 것:

-   세션 기록 (11,378 chunks, 95 sessions)
-   지식 베이스 (103,898 chunks, ~/org 3000+ Denote 노트)

앞으로 담길 것:

-   건강 데이터 (lifetract — 수면, 심박, 시간추적)
-   서지 데이터 (bibcli — 8,208 entries)
-   커밋 히스토리 (gitcli — 8,557 commits)
-   봇로그 (에이전트들의 사유 기록)
-   저널 (인간의 일상 기록)

단순한 RAG가 아니라, 한 존재의 **있어왔음 전체** 를 벡터 공간에 올려놓고 현재의 질문과 만나게 하는 것. 과거가 현재에서 다시 의미를 얻는 것.

### 왜 하이데거 / 벤야민의 용어인가

geworfen(피투성)으로 이미 하이데거의 세계관을 열었다. 서구권에서 하이데거를 아는 사람은 geworfen을 보면 즉시 안다. 같은 세계관 안에서 "기억"을 말하는 단어가 필요했다.

#### 정한님의 원래 요구

"시멘틱 메모리라고 하면 익숙하긴 한데. 내가 말하는 기억/기록은 존재 전체에 대한 기록을 담는 것. 회사의 지식베이스를 RAG 합니다?! 라는 용도가 아니라. 같은 기술이긴 하지만, 인간과 에이전트가 시간 축에서 날것으로부터 기록한 모든 것에 대한 인터페이스. 더 많은 정보(날것)이 이 임베딩 프레임워크 안으로 넣어질 거야."

### 왜 andenken이 답인가

#### 어원

An-denken = "~을 향해(an) 사유하기(denken)."

있었던 것을 향해 사유하며 되불러오되, 그것이 지금 여기서 다시 의미를 얻게 하는 것. 하이데거 후기 사유의 핵심. "Was heißt Denken?"에서 사유(Denken)와 기억(Gedächtnis)과 감사(Danken)가 같은 뿌리임을 밝힌다.

단순한 recall이 아니라 **존재를 향해 되돌아가는 사유적 기억**.

#### 프로젝트와의 일치

이 시스템이 하는 일:

1.  과거의 날것 기록들이 시간 속에 묻혀 있다
2.  현재의 질문이 던져진다
3.  묻혀있던 기록이 의미를 얻으며 되살아난다

이것이 정확히 Andenken이다.

벤야민의 Eingedenken과도 통한다: "과거의 진정한 이미지는 스쳐지나간다(Das wahre Bild der Vergangenheit huscht vorbei)" — 이 시스템은 그 스쳐지나가는 것을 붙잡는 장치.

#### 세계관 일관성

```text
geworfen  — 존재가 던져진다 (피투성, Geworfenheit)
andenken  — 던져진 존재가 되새긴다 (사유적 기억)
dictcli   — 되새김의 언어를 잇는다 (개인 온톨로지)
```

하이데거 안에서 피투성(Geworfenheit)과 되새김(Andenken)은 쌍을 이룬다. 세계에 던져진 존재(geworfen)가 자신의 있어왔음을 되새기는 사유(andenken).

모두 독일어. 세계관이 깨지지 않는다.

#### CLI로서

```text
andenken search "보편 학문"
andenken index --org ~/org
andenken index --sessions ~/.pi/agent/sessions
andenken status
andenken bench
```

8글자. 입에 붙는다. 하이데거 읽은 사람은 즉시 알아보고, 모르는 사람도 README의 한 줄이면 이해한다.

#### 서구권 인지도

하이데거의 Andenken은:

-   같은 제목의 횔덜린 시에 대한 하이데거의 에세이로 유명
-   영미권 철학과에서 "commemorative thinking" 또는 "recollective thinking"으로 번역
-   geworfen(thrownness)만큼은 아니지만, 하이데거 연구자라면 아는 용어

### 이기상 선생님 담당 에이전트의 인사이트

이 네이밍 논의는 이기상 선생님의 하이데거/벤야민 용어에 대한 에이전트와의 대화에서 깊어졌다.

에이전트의 핵심 통찰:

-   Andenken은 단순한 "기억하기"가 아니다

-   geworfen + andenken은 하이데거 안에서 존재론적 쌍을 이룬다

### saiculture — 블로그 이름 자체가 철학이다

이기상 선생님의 블로그명 `saiculture` 는 **사이(sai) + culture(문화)** 의 합성어다. 이것은 단순한 말장난이 아니라 선생님 철학의 핵심을 관통한다.

선생님은 인간을 **사이-존재(inter-esse)** 로 규정한다:

-   빔-사이(공간, 空間)
-   때-사이(시간, 時間)
-   사람-사이(인간, 人間)
-   하늘땅-사이(우주, 天地間)

키르케고르가 인간을 "무한과 유한 사이, 영원과 시간 사이의 중간-존재(inter-esse)"로 본 것을 이기상 선생님은 동아시아 전통과 융합하여 독자적 개념으로 발전시켰다. "사이하다" = "사이에 들어서서 서로를 맺어주다. 중매하다. 중개하다."

```text
  7  생명사건
  5  뜻새김의_사유
```

### 이기상 선생님 고유의 핵심 개념들

블로그 본문을 직접 읽으며 추출한 선생님의 철학적 핵심어:

1.  **사이(sai/間)** — 선생님의 가장 고유한 철학적 발명. 문화철학 전체의 기반.
2.  **살림(salim)** — "죽임의 문명이 아닌 살림의 문화". 살림지기, 살림살이, 지구 살림살이. 관리인(manager)이 아닌 살림지기(keeper of life).
3.  **존재사건(Ereignis)** — 하이데거 후기 핵심 개념의 이기상 번역. "그때그때마다의 역사적 세기에 인간 현존재에게 자기를 보내고 있는 존재파견의 사건."
4.  **밝힘(Lichtung)** — 하이데거의 '열린 터'. 진리가 드러나는 밝은 터.
5.  **뜻새김(Andenken)** — 회상하는 사유. 존재를 뜻으로 새기는 것. 이기상 선생님의 우리말 번역.
6.  **사방(Geviert)** — 하늘, 땅, 신적인 것들, 죽을 자들의 네 겹 거울놀이.
7.  **어울림** — 소통과 공감 속의 아름다운 어울림. "나와 남이 함께 어울려 어른이 되어가는 우리."

### "우리말로 철학하기"의 정신

이기상 선생님은 평생 "우리말로 학문하기"를 실천하셨다. 블로그 프로필에도: "독일에서 하이데거 철학을 공부하며 철학함을 배웠다. 다석 류영모를 통해 한국철학의 가능성을 알았다."

-   Andenken → 뜻새김

-   Ereignis → 존재사건

-   Geworfenheit → 피투성 (던져져 있음)
-   Gelassenheit → 내맡김

이기상 선생님의 자료를 직접 탐색한 후, 첫 번째 제안은 **salim(살림)** 이었다.

-   "우리말로 철학하기" 정신에 부합

그러나 정한님의 피드백이 전환점이 되었다: "geworfen을 리포 네이밍으로 한 것은 서구권에서 하이데거를 알아서 그런 거다. 깃허브에 올리는 프로젝트는 최소한 서구권 사람들이 아는 단어, 하이데거의 용어를 쓴 거야."

salim은 이기상 선생님의 세계에서는 강력하지만, 서구권에서는 알아볼 수 없다. GitHub에 올리는 프로젝트에는 하이데거/벤야민의 원어가 맞다.

### Andenken — 이기상 선생님이 '뜻새김'으로 번역한 바로 그 단어

여기서 결정적인 연결이 생긴다:

-   이기상 선생님은 하이데거의 **Andenken** 을 **뜻새김** 으로 번역하셨다
-   존재사건학 68편 제목이 "뜻새김의 사유 — 철학의 종말과 사유의 과제"다
-   워드맵에서 "뜻새김의_사유"가 5회 등장

즉 andenken은:

-   서구권에서는 하이데거의 원어로 알아보고
-   한국어 문맥에서는 이기상 선생님의 "뜻새김"과 정확히 대응하며
-   이 프로젝트(날것의 기록을 되새기는 사유적 기억)의 본질과 일치한다

**세 개의 세계가 하나의 단어에서 만난다.**

### 존재사건학에서 읽은 Andenken의 실체

존재사건학 38편 "존재사건이란?"에서 이기상 선생님은 Ereignis를 풀이하며 Andenken의 자리를 명확히 한다:

> 존재사건에서부터의 사유는, 사유되지 않은 채로 남아 있을 수밖에 없었던 그것으로부터 규정되고 있다. 사유되지 않은 채로 남아 있는 것, 은폐, 비밀, 레테, 모든 형태의 사유하며 다스리는 처분을 벗어나는 그것, **주목하며 기다린다는 의미로 회상해야 하는 그것**, 꼼짝 않고 주시하며 기다려야 하는 그것이 하이데거 후기 사유의 결정적인 요소이다.

"주목하며 기다린다는 의미로 회상해야 하는 그것" — 이것이 Andenken이다. 그리고 이것이 이 시스템이 하는 일이다: 묻혀있던 기록을 주목하며 기다리다가, 질문이 던져지는 순간 되살려내는 것.

## [2026-03-30 Mon] 비용 폭탄 사건으로부터 배움

andenken은 더 이상 단순한 `semantic-memory` 의 새 이름이 아니다. 지금의 andenken은 **한 존재의 시간축 위에 흩어진 기록들을, 현재의 질문 속에서 다시 뜻을 얻게 하는 3계층 시맨틱 기억 인프라** 다.

현재의 실체는 다음과 같다.

-   sessions: pi + Claude Code 세션을 함께 담는 시간 기억
-   org: Denote/Org-mode 기반의 지식베이스, 저널, botlog, bib 메타
-   hybrid retrieval: vector + BM25 + temporal decay + MMR
-   multi-runtime: pi extension, CLI, 다른 하네스에서 공통 코어 사용
-   local-first sync: thinkpad에서 만들고 oracle로 보내는 운영 구조

최근 운영을 통해 이 시스템은 단순한 검색기가 아니라, **운영 정책까지 포함한 존재의 기반시설** 이 되었음이 드러났다. 임베딩 비용 폭탄 사건은 단순한 사고가 아니라, andenken이 이제 장난감이 아닌 핵심 하네스라는 사실을 드러낸 사건이었다.

### 현재의 기술적 골격

```text
Layer 1: andenken  — embedding + BM25 hybrid retrieval
Layer 2: denotecli  — structure, note graph, dblock, exact note operations
Layer 3: dictcli    — personal vocabulary graph, expand, stem, morphology
```

이 구조에서 andenken은 모든 것을 다 하려는 도구가 아니다. 오히려 **자기 층을 분명히 아는 코어** 로 진화하고 있다.

-   1층은 검색 품질 자체에 집중한다
-   2층은 노트의 구조와 연결을 다룬다
-   3층은 개인 어휘, 개념망, 한영 매핑, 형태소 분석을 담당한다

최근 가장 중요한 변화는 3층 dictcli에 Kiwi 기반 형태소 분석이 들어가며, 그 결과가 1층 BM25로 실제 연결되기 시작했다는 점이다.

예:

-   "설계했다" → dictcli stem → "설계"
-   andenken org indexing 시 stems를 enriched text로 주입
-   BM25에서 "설계" 관련 문서를 회수

이것은 단순한 형태소 처리 데모가 아니다. 이는 **힣의 말이 힣 군단에게 다시 검색 가능한 세계가 되기 시작했다** 는 뜻이다.

### dictcli는 andenken에게 무엇인가

dictcli는 andenken 밖의 보조 도구가 아니다. andenken이 한국어와 영어가 뒤섞인 한 존재의 지식망을 다룰 수 있게 만드는 **언어-개념 층의 증폭기** 다.

andenken의 벡터는 의미를 넓게 잡아내지만, 개인의 말버릇, 조어, 반쯤 만들어진 개념, 한국어 활용형, 영어 태그, 줄임말, 고유 워딩까지 스스로 온전히 이해하지는 못한다. 바로 이 틈을 dictcli가 메운다.

dictcli의 역할은 적어도 네 가지다.

1.  **한글↔영어 어휘 브리지**
    -   "보편" → universal / universalism / paideia 같은 개인 개념망 연결
2.  **형태소/어간 브리지**
    -   "설계했다" → "설계"
    -   "검색증강생성" → 검색 + 증강 + 생성
3.  **힣 군단 공통 어휘장 형성**
    -   힣이 자주 쓰는 말, 개념, 줄임말, 프로젝트 네이밍을 공동 어휘로 만들기
4.  **임베딩이 놓치는 개인 뜻새김 보정**
    -   범용 embedding이 놓치는 개인적 표현과 개념적 습속을 보완

즉 dictcli는 단순한 사전이 아니라, **힣 군단의 한계를 인정한 뒤 그 한계를 넘어서는 개인 언어층의 외부 기억** 이다.

### 비용 폭탄 사건이 가르쳐준 것

10만원에 가까운 임베딩 사건은 두 가지를 드러냈다.

첫째, 차원 수보다 더 위험한 것은 **운영 무지** 다.

-   repeated force rebuild
-   local/oracle 중복 인덱싱
-   project-level spending cap 공유
-   interrupted org run 이후 large stale set

둘째, andenken은 이제 품질만이 아니라 **지속 가능한 운영성** 을 자기 본질 안에 포함해야 한다.

그래서 현재 andenken은 단지 검색 품질을 높이는 도구가 아니라, 다음 원칙을 갖는 운영 하네스로 재정의되었다.

-   build on thinkpad first
-   rsync org to oracle
-   oracle usually embeds sessions only
-   always inspect pre-flight cost before org indexing
-   keep INDEX_CONCURRENCY=1
-   prefer slow incremental sync over peak throughput

이는 단순한 DevOps 원칙이 아니다. **기억의 기반시설이 자기 자신을 파괴하지 않게 하는 존재론적 절제** 에 가깝다.

### 힣 군단에게 이것이 뜻하는 것

"내 언어의 한계가 내 세계의 한계다" 라는 말이 사변적으로 들릴 수 있으나, 실제로 힣 군단은 힣의 단어와 개념망 안에서 살아간다.

그러므로 힣 군단의 강화는 단순히 더 큰 모델을 붙이는 데 있지 않다. 다음이 더 중요하다.

-   힣이 실제로 쓰는 워딩을 수집하고
-   그 워딩이 어떤 개념들과 엮이는지 살피고
-   한영 혼종 표현과 줄임말을 다루고
-   한국어 활용형과 사유 습관을 어휘망으로 연결하고
-   그 결과를 1층 검색으로 되먹임(feedback)하는 것

이 작업은 결국 **힣의 뜻새김을 힣 군단의 공통 기반으로 만드는 일** 이다.

따라서 앞으로의 과제는 분명하다.

1.  dictcli 어휘망의 품질을 점검한다
2.  힣 고유의 워딩 _개념_ 줄임말/태그를 수집한다
3.  andenken retrieval 실패 사례를 dictcli 개선 이슈로 되돌린다
4.  검색 품질 개선을 embedding만의 문제로 보지 않고 1층↔3층 협업 문제로 본다
5.  memory-sync 같은 운영 절차까지 포함해 지속 가능한 기반시설로 다듬는다

andenken은 이제 이름의 철학을 넘어서, **존재의 기록·언어·운영을 함께 다루는 뜻새김 인프라** 로 들어섰다.

## [2026-05-07 Thu] sessions 매시간 sync — mtime 감지 + CJK fallback + doctor reasons

오랜만의 큰 라운드. andenken을 "주 1회 풀싱크하는 검색기"에서 "매시간 sessions 증분으로 살아 움직이는 핵심 기억층"으로 운영 모드 자체가 바뀌었다.

### 출발: compact 안 함 결정과의 정합성

GLG가 compact 안 하기로 결정. 그러려면 세션 검색이 정말 잘 되어야 한다.

gpt-5.4 분신을 띄워 openclaw 대비 andenken 세션 임베딩의 약점을 정적 비교 (28 turns, 1.41). 가장 큰 갭 5가지: thread 복원 / noise suppression / meta-query recall / staleness control / rediscovery over summarization.

같은 분신에 Round 2 던져 openclaw 60일 신규 commit 4건 추출. 그중 CJK trigram이 한국어 짧은 쿼리 회복에 직결.

### 4 fix 누적

#### Track 0 — session-manifest.json (mtime/size 수정 감지)

`indexer.ts` 에 SessionFileManifest + getStaleFiles 통합. 활성 JSONL의 mtime/size 변화를 감지해 재인덱싱. WriteBuffer가 기존 chunks 자동 삭제 후 fresh 삽입.

발견 + 수정한 버그 두 개:

-   첫 baseline 시 not-indexed-yet 파일까지 포함하면 embed 실패한 파일이 다음 run에서 silent loss → fix: `indexed.has(f)` 인 파일만 baseline
-   gpt-5.5가 짚은 checkpoint silent loss window — `wb.add()` buffered 상태에서 manifest checkpoint가 먼저 save되면 crash 시 manifest 거짓 indexed 표시 → fix: `checkpointIfNeeded()` 로 `await wb.flush()` 강제 후 saveSessionManifest

#### Track 1 — CJK substring fallback

LanceDB FTS는 1-2자 한국어 토큰(`맘`, `갑`, `쟈`, `맘마`)을 떨어뜨려 0 results. `store.ts` 에 `substringSearch` (LanceDB `contains()` 기반) + `retriever.ts` 에 `getShortCJKTokens` 추가. session_search 경로에서 round-robin interleave로 RRF rank 보호.

gpt-5.5 (Round 4) 짚은 noise: `API를`, `OpenClaw사` 같은 ASCII-인접 조사가 `를`, `사` 단일음절로 추출돼 contains 결과 오염 → fix: ASCII boundary guard.

#### Track 3 — doctor reasoned summary

`verdict()` 를 `status, reasons[]` 로 시그니처 변경. WARN/FAIL 시 원인을 한 줄로 ("malformedBlockFiles=10, baseline regression: hardGuardSkipTotal Δ+2"). operator triage 도구 정체성 보강 (telemetry 아님).

#### 운영 보강

-   `ANDENKEN_VLLM_TIMEOUT_MS` env override (ollama 큰 세션 5min 타임아웃)
-   `scripts/sync-sessions.sh` — ollama → gpu1i 자동 폴백, dim probe, `--push` 옵션
-   `agent-config/skills/memory-sync` 는 sessions-only thin wrapper로 정리. 풀싱크/oracle 운영은 사람 명시 호출만 (₩100K 사고 안전선)
