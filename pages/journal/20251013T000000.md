### <span class="org-todo done DONE">DONE</span> 12:06 유니코드 한글 로케일 이슈

```markdown

● 조사 결과: 터미널 이맥스 한글 입력 유니코드 변환 문제

  🔍 문제 분석

  사용자가 겪는 "엸\260" 형태의 문제는 Octal Escape Sequence 표시 문제입니다. 이맥스가 멀티바이트 문자(CJK 포함)를 제대로
  해석하지 못할 때 128-255(0200-0377 octal) 범위의 바이트를 이스케이프 코드로 표시합니다.

  🎯 주요 원인 (CJK 공통)

  1. 로케일/인코딩 미스매치 (가장 흔한 원인)

  # 현재 로케일 확인
  echo $LANG
  echo $LC_CTYPE

  # UTF-8이 아니거나 설정되지 않았다면 문제 발생

  2. 터미널 코딩 시스템 미설정

  이맥스가 터미널의 인코딩을 올바르게 인식하지 못함

  3. 유니코드 정규화 문제 (특히 macOS)

  - macOS는 기본적으로 NFD 정규화 사용
  - 한글이 자음+모음으로 분해됨: "가" → "ㄱ + ㅏ"
  - 터미널이 분해된 코드포인트를 처리하지 못함

  4. 멀티바이트 모드 비활성화

  버퍼가 unibyte 모드로 되어 있을 때

  ✅ 해결 방법

  방법 1: 환경변수 설정 (터미널 시작 시)

  # ~/.bashrc 또는 ~/.zshrc에 추가
  export LANG=ko_KR.UTF-8
  export LC_ALL=ko_KR.UTF-8
  export LC_CTYPE=ko_KR.UTF-8

  방법 2: 이맥스 설정 (init.el)

  ;; UTF-8 환경 강제 설정
  (set-language-environment "Korean")
  (prefer-coding-system 'utf-8)
  (set-default-coding-systems 'utf-8)
  (set-terminal-coding-system 'utf-8)
  (set-keyboard-coding-system 'utf-8)

  ;; 멀티바이트 모드 활성화 (필요시)
  (set-buffer-multibyte t)

  방법 3: 로케일 생성 (Ubuntu/Debian)

  # 로케일이 시스템에 없을 경우
  sudo locale-gen ko_KR.UTF-8
  sudo dpkg-reconfigure locales

  방법 4: 실시간 수정 (현재 세션)                                                                                                                                             M-x set-terminal-coding-system RET utf-8 RET                                          M-x set-keyboard-coding-system RET utf-8 RET
  M-x set-language-environment RET Korean RET

  🔧 즉시 테스트                                                                                                                                                              # 1. 터미널에서 로케일 확인                                                           locale

  # 2. 이맥스 실행 시 환경변수 명시
  LANG=ko_KR.UTF-8 emacs -nw                                                                                                                                                  # 3. 이맥스 내에서 인코딩 확인                                                        M-x describe-coding-system RET

  📚 참고: CJK 언어 공통 이슈

  - 중국어/일본어도 동일한 문제 발생
  - 멀티바이트 문자 처리는 CJK 전체에 공통적인 기술적 도전
- 터미널 에뮬레이터 종류에 따라 문자 폭 인식 차이도 존재
  - 특정 유니코드 범위는 char-width-table 수정 필요할 수도 있음

  🎯 추천 조치
  1. 먼저 로케일 확인: locale 명령어로 UTF-8 설정 확인
  2. 환경변수 영구 설정: ~/.bashrc에 LANG=ko_KR.UTF-8 추가
  3. 이맥스 설정 추가: init.el에 UTF-8 코딩 시스템 명시
  4. 문제 재현 시: 이맥스 내에서 C-u C-x = (what-cursor-position)로 문자 정보 확인

  이 문제는 로케일 설정 + 이맥스 코딩 시스템 설정으로 대부분 해결됩니다.
```

#### 세가지 모드

```markdown
> A공유기에서 포트중에 하나를 B공유기의 WAN에 연결하는 방식으로 인터넷을
확장하는게 가능한가?

● English translation of question: "Is it possible to extend the internet by
  connecting one of the ports from Router A to the WAN port of Router B?"

  네, 가능합니다. 여러 방식이 있습니다:

  1. 라우터 모드 (Router Mode) - WAN 연결

  A공유기(192.168.0.1) LAN 포트 → B공유기 WAN 포트
                                    └─ B공유기(192.168.1.1) LAN 포트들

  특징:
  - ✅ 가장 간단 (B공유기 WAN에 그냥 연결)
  - ❌ 이중 NAT 발생 (네트워크 분리)
  - A 네트워크: 192.168.0.x
  - B 네트워크: 192.168.1.x
  - B에서 A 장치 접근 어려움

  설정: B공유기를 라우터 모드로 두고 WAN에 연결

  2. AP 모드 (추천)

  A공유기(192.168.0.1) LAN 포트 → B공유기 LAN 포트 (WAN 사용 안 함)
                                    └─ 같은 네트워크(192.168.0.x)

  특징:
  - ✅ 모든 장치가 같은 네트워크
  - ✅ 장치 간 통신 자유로움
  - ✅ 단일 DHCP 서버

  설정:
  1. B공유기 설정에서 "AP 모드" 또는 "브리지 모드" 선택
  2. B공유기 DHCP 끄기
  3. B공유기 IP를 192.168.0.2로 변경 (충돌 방지)
  4. A공유기 LAN → B공유기 LAN 포트 연결

  3. 스위치 모드

  - B공유기 라우터 기능 완전히 끄고 스위치로만 사용
  - AP 모드와 유사하지만 WiFi도 없음

  현재 문제와의 연관성:
  지금 enp1s0f0가 연결된 포트가 혹시 B공유기의 WAN 포트인가요? 그렇다면 B공유기가
  아직 인터넷을 받지 못해서 DHCP가 작동하지 않을 수 있습니다.

  어떤 구성을 원하시나요?

```
