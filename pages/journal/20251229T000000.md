#### Magit에서 Git Log 검색 방법

**기본 사용법:**

1.  `l` - magit-log transient 메뉴 열기
2.  검색 필터 추가:
    -   `-G` : Changes(변경사항) 검색 (정규식 기반)
    -   `-S` : Occurrences(발생) 검색 (pickaxe - 문자열 출현 횟수 변화 추적)
    -   `-F` : Commit Message(커밋 메시지) 검색 (`--grep`)

**주요 차이점:**

-   **`-G <regex>`**: 정규식으로 변경사항을 검색합니다. 코드 내용의 변화를 찾을 때 유용합니다.
-   **`-S <string>`**: 특정 문자열의 추가/삭제를 추적합니다 (Git pickaxe). 예를 들어 함수 이름이나 변수가 언제 추가/제거되었는지 찾을 때 사용합니다.
-   **`-F --grep=<pattern>`**: 커밋 메시지에서 패턴을 검색합니다.

**예시 워크플로우:**

```nil
l          ; log 메뉴 열기
-G         ; -G 옵션 활성화하고 검색어 입력
l          ; log 보기 (또는 다른 log 변형 선택)
```

이 기능들은 Magit의 transient 인터페이스를 통해 제공되며, Git의 강력한 로그 검색 기능을 Emacs 환경에서 편리하게 사용할 수 있게 해줍니다.

Question: How can I browse files based on a specific branch? I need a way to view file contents from around that time period.

Magit에서 특정 브랜치나 커밋 시점의 파일을 브라우징하는 방법을 알아보겠습니다.네, Magit에서 특정 브랜치나 커밋 시점의 파일들을 브라우징하는 방법이 여러 가지 있습니다!

##### 1. Log 버퍼에서 직접 탐색

가장 직관적인 방법:

```nil
l l        ; log 버퍼 열기
SPC        ; 커밋 상세정보 표시 (현재 버퍼 유지)
C-n/C-p    ; 위아래 이동하면 자동으로 커밋 내용 업데이트
RET        ; 특정 diff 라인에서 해당 파일의 blob 열기
```

-   diff에서 /추가/컨텍스트 라인\*에 커서를 놓고 `RET` → 새 버전(new/right) 파일 열림
-   diff에서 /삭제된 라인\*에 커서를 놓고 `RET` → 이전 버전(old/left) 파일 열림

##### 2. magit-find-file 명령어

특정 리비전의 파일을 직접 열기:

```nil
M-x magit-find-file
```

이 명령은:

1.  리비전(커밋, 브랜치, 태그)을 입력받고
2.  파일명을 입력받아
3.  해당 시점의 파일 내용을 버퍼로 엽니다

4.  `magit-find-file-other-window` : 다른 윈도우에 표시

##### 3. git-timemachine 패키지 (보조 도구)

특정 파일의 히스토리를 시간여행하듯 탐색:

-   MELPA에서 설치: `git-timemachine`
-   현재 파일의 과거 버전들을 `C-n/C-p` 로 쉽게 이동하며 확인 가능

#### 실무 워크플로우 예시

특정 시점의 여러 파일을 확인하고 싶을 때:

```text
1. l l                    ; 로그 보기
2. -G "검색어"            ; 원하는 시점 찾기
3. 해당 커밋으로 이동
4. RET                    ; 커밋 상세 보기
5. 각 파일의 diff에서 RET ; blob 버퍼 열기
```

또는:

```text
1. M-x magit-find-file
2. 브랜치명/커밋 해시 입력 (예: feature/user-auth 또는 HEAD~5)
3. 파일 경로 입력
4. 해당 시점의 파일 내용 확인
```

_참고:_ 현재 Dired처럼 트리 구조로 브라우징하는 네이티브 기능은 Magit에 없지만, 위 방법들로 충분히 특정 시점의 파일들을 탐색할 수 있습니다.
