#### 1장 자연과학의 연구 방법

##### 1. 여는 말

지식은 인간에 의해 만들어진다. 이 지식이 인간의 생존에 얼마나 중요한 역할을 하는가에 대해서는 이 자리에서 굳이 깊이 논의할 필요는 없을 것이다. 그보다는 오히려 지식이 인간에 의해 어떻게 만들어지는가를 아는 것이 우리에게는 더 큰 관심사가 될 것이다. 흥미롭게도 우리는 일상생활을 통해 자기도 의식하지 못하는 가운데 적지 않은 소중한 지식들을 만들어가고 있다. 이렇게 만들어진 지식을 우리는 '일상적 지식'이라 부른다. 그런데 이것에 대비되는 또 한 가지 지식이 있다. 이른바 '과학적 지식'이다. 일상적 지식과는 달리 과학적 지식은 무의식 속에서 손쉽게 얻어지는 것이 아니다. 적어도 과학 공부라고 하는 학습과정을 통해서, 그리고 과학 연구라고 하는 좀 더 높은 수준의 정신적 활동을 통해서 만들어지는 것이다.

대부분의 경우 우리는 특별한 학습방법론이나 연구방법론에 대한 이해 없이 과학 공부를 시작하며 또 과학 연구에 뛰어든다. 그러다가 학습과 연구 과정을 통해 자기도 모르는 사이에 일정한 학습 방법이나 연구 방법을 체득하게 된다. 우리는 과학 지식을 습득하고 과학 지식을 창출해내지만 이를 습득하고 창출하는 과정은 말하자면 전과학적인 방법에 의존하는 셈이다

그렇다면 과학의 학습과 연구를 위한 과학적 방법은 없는 것 인가? 안타깝게도 아직까지 어떤 공인된 방법은 없다 그러나 적지 않은 사람들이 이에 대해 줄곧 고민해왔고 또 공개적인 논의가 이루어진 것 또한 사실이다. 그러므로 이 장에서는 특히 과학 연구 방법과 관련한 논의들을 몇 가지 기준에 맞추어 정리 해보고 나름대로의 대안을 제시해보고자 한다

##### 2. 연구방법론의 두가지 측면

자연과학을 비롯한 모든 학문의 연구방법론은 두 가지 측면이 서 고찰해볼 수 있다. 그 하나는 어떠한 방법으로 학문 연구를 해야 하느나는 규범적 측면에서의 고찰이며, 다른 하나는 실제 로 학자들이 어떠한 방법으로 학문 연구를 하고 있느냐는 현실적 측면에서의 고찰이다. 학문 연구 당사자로서는 당연히 자신 들이 어떠한 방법으로 학문 연구를 해나가야 할 것인가 하는 규범적 의미에서의 방법론이 주된 관심의 대상이 될 것이며 따라서 방법론은 마땅히 규범적 의미를 지녀야 할 것이다. 하지만 학문 발전의 양상 자체에 관심을 갖는 객관적 입장에서는 학문 연구 당사자들이 현실적으로 어떠한 방식에 의존해 학문 연구를 해나가는가에 더 큰 비중을 두게 될 것이며 따라서 학문 연구 방법의 현실적 측면 또한 중요한 의미를 지니게 된다. 만일 이 두가지 측면에서 고찰된 방법들이 서로 일치하지 않을 경우 이는 기존의 방법론 자체에 문제점이 있다는 것을 의미하게 되며, 다시 이를 극복하기 위한 더 고차적인 방법론이 모색되어야 할 것이다.

```markdown
● SSH 키 선택 문제 용어

  일반적인 명칭:
  - SSH Key Selection Issue / SSH 키 선택 문제
  - SSH Agent Key Priority Problem / SSH Agent 키 우선순위 문제
  - SSH Identity Confusion / SSH 신원 혼동

  더 구체적으로:

  키가 사용됨

  기록용 요약

  ## SSH 키 선택 문제 (SSH Identity Confusion)

  ### 증상
  - 여러 GitHub 계정 사용 시 잘못된 계정으로 인증됨
  - `Permission denied to <wrong-user>` 오류

  ### 원인
  1. ssh-agent에 여러 키 로드됨
  2. GitHub는 첫 번째 유효한 키로 인증
  3. ControlMaster가 기존 연결 캐싱

  ### 해결
  1. ~/.ssh/config에 `IdentitiesOnly yes` 추가
  2. 기존 연결 초기화: `ssh -O exit <host>`

  검색 키워드: ssh multiple github accounts IdentitiesOnly
```

#### Blur: How to Know What's True in the Age of Information Overload

**질문(영문):** Tell me about the book "Blur: How to Know What's True in the Age of Information Overload"

---

##### 책 개요

**저자:** Bill Kovach &amp; Tom Rosenstiel **출간:** 2010년 **장르:** 미디어 리터러시(Media_Literacy), 저널리즘 이론

##### 핵심 메시지

정보 과부하 시대에 \*무엇이 진실인지 판별하는 방법\*을 다룹니다. 전통 저널리즘의 검증 원칙을 일반 시민에게 확장하여, 누구나 정보 소비자이자 생산자가 된 디지털 시대의 \*비판적 사고 프레임워크\*를 제시합니다.

##### 주요 내용

###### 1. 정보의 홍수와 진실의 희석

-   소셜 미디어, 블로그, 즉각적 뉴스 사이클로 \*신뢰도 검증 없는 정보\*가 범람
-   "Blur" 현상: 사실(Fact), 의견(Opinion), 선전(Propaganda)의 경계가 흐려짐

###### 2. 6가지 질문 프레임워크

정보를 만날 때 스스로 묻는 질문들:

1.  **누가 이것을 만들었나?** (출처의 신뢰성)
2.  **증거는 무엇인가?** (데이터, 인용의 검증 가능성)
3.  **다른 해석은 없나?** (편향 체크)
4.  **빠진 것은 무엇인가?** (맥락, 반대 의견)
5.  **왜 이 정보가 나에게 왔나?** (동기, 알고리즘)
6.  **신뢰할 만한가?** (패턴, 과거 기록)

###### 3. 시민 저널리스트의 책임

-   누구나 정보를 퍼뜨릴 수 있는 시대 → **검증 의무도 분산**
-   전문 저널리스트의 윤리 기준을 일상적 정보 소비에 적용

##### 연결 포인트: 당신의 관심사와

###### Digital Garden &amp; Denote

-   Zettelkasten 방식으로 정보를 수집할 때, 이 6가지 질문을 \*노트 템플릿 체크리스트\*로 활용 가능
-   예: `#+FILETAGS: :source_verified:counterargument_needed:`

###### Being to Being with AI

-   AI가 생성한 정보도 "누가 만들었나?(모델 버전, 학습 데이터)", "빠진 것은?(환각, 편향)" 질문 적용
-   Claude와의 대화에서 \*출처 요구, 반론 제시 요청\*은 이 프레임워크의 실천

###### 메타휴먼 지향

-   1KB 정수로서의 자기 정의 ← \*노이즈 제거 능력\*이 전제
