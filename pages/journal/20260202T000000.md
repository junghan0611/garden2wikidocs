#### 핵심 아이디어 분석

##### 제안서 작성 워크플로우의 핵심 문제

1.  **다수 작성자 통합**: 여러 세부과제 담당자들의 문서를 하나로 병합
2.  **일관성 확보**: 용어, 톤앤매너, 양식의 통일
3.  **정부 양식 호환**: HWPX 포맷 + 매크로 지원
4.  **AI 편집 용이성**: 에이전트가 개입하여 자동 정리

##### Org-mode 메타 포맷의 강점

```org
*** 연구 목표
:PROPERTIES:
:FONT: 맑은고딕
:SIZE: 11pt
:INDENT: 0
:END:

**** 세부 목표 1
:PROPERTIES:
:TERM_MAPPING: "인공지능" → "AI시스템"
:STYLE: 공공언어
:END:
```

이 방식은:

-   구조화된 메타데이터로 양식 제어
-   용어 매핑 테이블 내장 가능
-   에이전트가 PROPERTIES 읽고 일관성 검증/적용

##### §memex-kb 확장 제안

###### 1. 제안서 전용 레이어 추가

```org
*** §memex-kb 구조
**** 기존: notes → denote → digitalgarden
**** 신규: notes → proposal-layer → hwpx
   - proposal-layer: 제안서 메타포맷
   - 용어사전, 스타일가이드 임베딩
   - 다중 org 파일 병합 로직
```

###### 2. 구체적 기능 설계

**A. 용어 통일 시스템**

```org
#+TERM_DICTIONARY:
| 원문 | 통일용어 | 적용범위 |
|------+----------+----------|
| AI   | 인공지능 | 전체     |
| ML   | 머신러닝 | 기술섹션 |
```

**B. 스타일 가이드**

```org
#+STYLE_GUIDE:
- 공공언어 규칙 적용
- 문장 길이: 최대 80자
- 한자 병기: 첫 등장 시만
- 외래어: 순화어 우선
```

**C. 병합 메타데이터**

```org
#+MERGE_SOURCES:
- [[file:세부과제1.org][세부과제 1: 데이터 수집]]
- [[file:세부과제2.org][세부과제 2: 모델 개발]]

#+MERGE_STRATEGY:
- 헤딩 레벨 조정: auto
- 각주 번호 재정렬: yes
- 참고문헌 통합: bibtex
```

###### 3. HWPX 변환 경로

```mermaid
graph LR
A[여러 .org 파일] --> B[proposal-layer 병합]
B --> C[용어/스타일 통일]
C --> D[Pandoc + 커스텀 필터]
D --> E[HWPX 출력]
E --> F[정부 양식 매크로 적용]
```

**기술적 고려사항:**

-   Pandoc의 HWPX writer는 제한적 → 커스텀 Lua 필터 필요
-   또는 Org → LaTeX → HWPX 경로 (hwpx-converter 도구 활용)
-   매크로는 템플릿 HWPX에 미리 내장 후 내용만 채우기

##### 실행 가능한 프로토타입

###### Phase 1: 개념 증명

1.  간단한 org 2개 병합
2.  용어 매핑 테이블 적용
3.  스타일 통일 (문장 길이, 어투)
4.  HTML 출력으로 확인

###### Phase 2: HWPX 변환

1.  Pandoc 기반 변환 테스트
2.  정부 양식 템플릿 분석 (\*.hwp → HWPX 구조)
3.  매크로 보존 방법 연구

###### Phase 3: AI 에이전트 통합

1.  LLM이 org PROPERTIES 읽어 검증
2.  일관성 검사 (용어, 톤, 구조)
3.  자동 수정 제안
