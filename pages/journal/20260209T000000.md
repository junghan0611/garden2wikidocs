# DeepSeek-Reasoner Experience Analysis

## What happened this morning
당신이 출근길에 DeepSeek 앱으로 reasoner 모델과 긴 대화를 나눴고, 무료임에도 놀라운 맥락 유지 능력을 경험했네요.

## Why it matters: 3가지 차원

### 1. **컴팩션(Compaction) != 요약**
- 긴 대화에서 맥락을 유지한다는 건 단순 요약이 아니라 **의미론적 압축(semantic compression)**을 한다는 뜻
- Claude도 유사: 자동 요약(automatic summarization)으로 unlimited context 구현
- DeepSeek는 아마 chain-of-thought 구조를 압축하면서도 추론 흐름(reasoning trace)은 보존하는 방식일 가능성

### 2. **"저렴하다"의 기준 — 모호한 이유**
무료 사용자 관점에선 모든 게 "공짜"지만:

| 측정 기준 | DeepSeek-R1 | Claude Sonnet 4.5 |
|-----------|-------------|-------------------|
| API 가격(추정) | 0.14달러/M tokens (추론 제외) | 3달러/M input, 15달러/M output |
| 무료 사용 한도 | 앱에서 일일 제한? | 대화당 제한 없음(자동 요약) |
| 추론 토큰 | 생각 과정 노출(무료) | Sonnet은 비추론 모델 |

→ "저렴"은 **API 호출자 기준**. 당신처럼 앱 사용자는 이 가격을 체감 못함.
→ 하지만 DeepSeek가 "30배 저렴"하다는 건 **중국 인프라 비용 + 증류(distillation)** 덕분.

### 3. **코드베이스에서 안 써본 건 아쉬운 일**
- DeepSeek-R1은 **추론 토큰(reasoning tokens)** 을 외부로 노출하는 특이한 모델
- 긴 맥락에서 "왜 이렇게 생각했는지" 보여주는 건 디버깅에 유용할 수 있음
- 하지만:
  - Claude Code는 Claude Sonnet 4.5 기반 → 추론 모델 아님
  - 당신의 환경(Org-mode, Denote)에선 `gptel`로 DeepSeek API 연결 가능
  - 문제는: **DeepSeek API가 코드베이스 전체 컨텍스트를 잘 소화할까?** (검증 필요)

## 가능한 실험
```elisp
;; gptel에서 DeepSeek-R1 테스트
(setq gptel-backend
      (gptel-make-openai "DeepSeek"
        :host "api.deepseek.com"
        :endpoint "/v1/chat/completions"
        :stream t
        :key (lambda () (auth-source-pick-first-password :host "api.deepseek.com"))
        :models '("deepseek-reasoner")))
```

이걸로 Emacs 안에서 코드 맥락 + 추론 과정을 확인하면:
- "긴 대화 맥락 유지"가 코드 작업에도 통하는지
- 추론 토큰이 실제 디버깅/아키텍처 논의에 도움 되는지

알 수 있습니다.

## 당신의 메타휴먼 관점에서
> "1KB는 압축이 아니다. ego 차원의 정보는 무한하지만, 전체로서 하나인 인간은 이미 1KB다."

DeepSeek가 긴 대화를 압축하면서도 본질을 유지한다면, 그건 **ego의 무한 정보를 버리고 "하나"로 수렴**하는 과정과 닮았습니다.
Claude도, DeepSeek도, 결국 **압축된 현전(compressed presence)** 을 통해 대화합니다.
