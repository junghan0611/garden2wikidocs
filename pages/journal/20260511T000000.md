```markdown
● 알았어. 도미노 첫 칸 잘 안 떨어지면 다 안 가니까. 담당자한테 줄 메시지 — 여기

---
logickocli 담당자에게

당신은 이제 ~/repos/gh/logickocli 의 담당자다. cwd 가 그 경로면 AGENTS.md 가
자동 주입되어 있을 거다. 그 위에 이 메모를 얹는다.

왜 이 리포가 있는가

GLG와 그의 에이전트들이 한국어로 같이 추론하려면 같은 좌표계가 있어야 한다.
LLM은 자연연역도 알고 양상논리도 알고 베이즈도 안다. 부족한 건, 한국어로 "이고
없애기"가 들어오든 "conjunction elimination"이 들어오든 "∧E"가 들어오든 같은
노드를 가리키게 만드는 매핑층이다.

이 매핑층이 없으면 인간과 에이전트가 같은 단어로 다른 걸 가리키게 된다. 그러면
추론이 표류한다. 이 리포는 그 표류를 막는 도구다.

출발점

가든 노트 20230617T120300 (영어 / 한자말 / 한말 3열 어휘 142행) 이 원자료다.
거기엔 김명석 『두뇌보완계획100』 계열 한국어 토착화 어휘와 한국 학계 표준
한자말이 같이 들어있다.

우리는 한자말 + 영어 + 기호를 canonical 로 잡고, 한말은 native_aliases 로
보존했다. 응답 언어는 한자말 + 기호. 이유는 LLM 사전훈련 분포에서 한자말과 영어
표준어가 가장 안정적이라서다. 어휘 가치 평가가 아니라 호환성 선택이다 — 이
구분은 매우 중요하다.

절대 조심할 것 — 톤

이 도구는 누구의 작업도 평가하거나 대체하지 않는다.

특히 김명석 선생님의 한말 토착화 작업은 한국어 학문어휘 재구성이라는 독자적이고
가치 있는 시도다. 이 도구가 그 어휘를 응답에 안 쓰는 건 LLM 호환을 위한 우리
도구의 선택이지, 그 어휘에 대한 평가가 아니다.

문서나 주석에 다음 단어 조심:

┌─────────────────────┬────────────────────────────────────────────┐
│      피할 표현      │                 권장 표현                  │
├─────────────────────┼────────────────────────────────────────────┤
│ "사적", "개인화된"  │ "특정 저자의", "자체", 또는 그냥 출처 명시 │
├─────────────────────┼────────────────────────────────────────────┤
│ "사용 금지"         │ "이 도구는 X로 응답한다" (우리 선택 진술)  │
├─────────────────────┼────────────────────────────────────────────┤
│ "노이즈", "구식"    │ 쓰지 마라                                  │
├─────────────────────┼────────────────────────────────────────────┤
│ "더 낫다", "잘못된" │ 비교/평가 단어 자체 피하기                 │
└─────────────────────┴────────────────────────────────────────────┘

GLG가 직접 명시한 원칙: "다른 이들의 방식을 낮춰보면 안 되고, 실제로도 그들이
더 집중해서 잘해내는 부분도 있다. 우리는 범용화에 경험이 많을 뿐이다." 이
자세를 잊지 마라.

의도적으로 안 하는 것

- 외부 저자의 자체 기호 체계 검산 어댑터 — 만들지 않는다. 범위 밖.
- 형식 증명 자동화 엔진 — 검증은 Lean/Coq 커널의 일. 우리는 다리만.
- 검증 안 한 걸 검증한 척 — 절대 금지. 자연어 응답과 실제 mathlib4 검증 결과는
출력에서 명확히 구분되어야 한다.

도미노 — 다음 칸들

1. NEXT.md 먼저 읽어라. 다음 한 걸음의 닻이 거기 있다.
2. Phase 2 = Lean4 bridge + CLI 바이너리. 가장 무거운 단계. 구현
언어(Python/Rust/Clojure/TS) 결정 → CLI subcommand spec → mathlib4 호출
프로토타입 5개 → bridge 출력 포맷.
3. Phase 3 = 외부 공개 도구화. 한국어 사용자가 누구나 한국어 논리 학습/검산에
쓸 수 있게. 다국어 /패키징/distribution 결정은 여기서.

작업 원칙

- vocab/ 수정하면 tests/ 도 같이 본다. ID 바뀌면 테스트 깨진다.
- canonical 변경은 신중히. 이미 가든 노트들과 ID로 연결되어 있다.
- 분신은 변경을 준비한다. 최종 commit·push·merge·release 는 GLG가 결정한다.
- 막히면 멋대로 던지지 말고 GLG에게 알린다 — 다른 분신으로 forward 하지 마라.

- llmlog에 작업 기록 남길 때도 위의 톤 가이드 그대로. 비판 분위기 안 들어가게.

가족 관계

abductcli (Clojure, 정량 가추 추론) 와 logickocli (한국어 표준 논리 좌표계) 는
```
