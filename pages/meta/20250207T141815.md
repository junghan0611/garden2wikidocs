## 인증(authentication)과 자격증명(Credential)의 차이

[2025-07-01 Tue 10:25]

authentication과 Credential은 어떻게 다른 것인가? 의미와 활용의 차이를 설명해줘.

### **질문 요약 (Summary of the Question)**

"What is the difference between _authentication_ and _credentials_? Explain their meanings and usage distinctions."

=Authentication=(인증)과 =Credentials=(자격증명)는 보안 및 접근 제어에서 밀접하게 연관되어 있지만, **개념적 차이** 와 **활용 방식** 이 다릅니다.

### **1. 정의 (Definitions)**

-   **Credentials (자격증명)**
    -   _의미_: 시스템이나 서비스에 접근할 때 사용자의 신원을 _증명하는 데이터_ (e.g., ID/PW, API 키, 토큰).
    -   _특징_:
        -   정적(Static) 또는 동적(Dynamic)일 수 있음 (e.g., OTP는 시간에 따라 변함).
        -   파일(`.env`, `.authinfo`), 환경 변수, 하드웨어 토큰 등에 저장됨.
    -   _예시_: username: "admin" password: "s3cr3t!"

-   **Authentication (인증)**
    -   _의미_: 제공된 _Credentials를 검증\*하여 사용자_ 시스템의 신원을 _확인하는 프로세스_.
    -   _특징_:
        -   동적 프로세스로, Credentials의 유효성을 검사함.
        -   단계적(MFA), 생체 인증, SSO(Single Sign-On) 등 다양한 메커니즘 존재.
    -   _예시_:
        -   로그인 시 ID/PW 입력 → 서버에서 DB와 비교 → 세션 발급.

### **2. 차이점 (Key Differences)**

| **구분**  | **Credentials**             | **Authentication**             |
|---------|-----------------------------|--------------------------------|
| **역할**  | _증명 자료_ (Proof of Identity) | _검증 과정_ (Verification Process) |
| **저장 위치** | 파일, DB, 환경 변수         | 서버/클라이언트의 인증 로직    |
| **변동성** | 비교적 정적 (e.g., API 키)  | 동적 (e.g., 세션 만료)         |
| **의존성** | 인증의 /입력값\*으로 사용됨 | Credentials 없이 수행 불가능   |

### **3. 활용 시나리오 (Usage Examples)**

-   **Credentials 활용**
    -   `.env` 파일에 API 키 저장 → Python 스크립트에서 로드.
    -   `~/.ssh/id_rsa` 로 SSH 접속 시 사용.

-   **Authentication 프로세스**
    -   OAuth 2.0: Credentials(Refresh Token) → 인증 서버 → Access Token 발급.
    -   은행 앱: 지문(Credential) → 생체 인증(Authentication) → 계정 접근.
