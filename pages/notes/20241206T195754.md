## 2024-12-06

### GIT LFS

@user Documents 디렉토리를 Git 으로 관리하려는데 LFS 로 관리하는 방법 및 그 장점에 대해서 알려줘

@assistant Git LFS (Large File Storage)를 사용하여 `Documents` 디렉토리를 관리하는 방법과 그 장점은 다음과 같습니다:

,\*\*\*\* Git LFS 설치 및 설정

1.  **Git LFS 설치**:
    ```bash
    git lfs install
    ```

2.  **LFS 트래킹 설정**:

트래킹할 파일 확장자 지정

```bash
git lfs track "*.jpg"  "*.png" "*.gif" "*.mp4"
# "*.epub" "*.pptx" "*.pdf" "*.docx"
```

1.  **.gitattributes 파일 확인**:
    ```bash
    cat .gitattributes
    ```
    이 파일에 트래킹할 파일 유형이 추가되었는지 확인합니다.

2.  **Git 초기화 및 커밋**:
    ```bash
    git init
    git add .
    git commit -m "Initial commit with LFS"
    ```

,\*\*\*\* 장점

-   **대용량 파일 효율적 관리**: Git LFS 는 대용량 파일을 효율적으로 저장하고 전송합니다. 실제 파일 대신 포인터만 저장되어 Git 저장소의 크기를 줄입니다.

-   **성능 향상**: 대용량 파일이 많을 때, 일반 Git 보다 훨씬 빠르게 클론, 페치, 푸시 작업을 수행할 수 있습니다.

-   **버전 관리**: LFS 를 사용해도 파일의 버전 관리는 여전히 가능합니다. 파일이 변경될 때마다 새로운 버전이 저장됩니다.

-   **저장소 크기 관리**: LFS 서버에 파일을 저장하므로, Git 저장소 자체의 크기를 줄일 수 있습니다.

-   **협업 용이**: 팀원들이 대용량 파일을 다운로드할 필요 없이 필요한 파일만 선택적으로 다운로드할 수 있습니다.

,\*\*\*\* 주의사항

-   **LFS 서버 필요**: Git LFS 는 별도의 서버가 필요합니다. GitHub, GitLab 등의 서비스에서 무료로 제공되지만, 용량 제한이 있을 수 있습니다.
-   **추가 설정 필요**: LFS 를 사용하려면 Git 설정과 함께 LFS 서버 설정도 필요합니다.
-   **비용**: 대용량 파일을 많이 저장할 경우, LFS 서버의 저장 공간 비용이 발생할 수 있습니다.

이러한 방법으로 `Documents` 디렉토리를 Git LFS 로 관리하면, 대용량 파일을 효율적으로 버전 관리할 수 있습니다.
