1.  **초기 설정**: 대화가 시작될 때, 각 메시지의 시작과 끝 위치가 기록됩니다. 예를 들어, `(498 . 971)` 는 첫 번째 메시지가 498번째 문자에서 시작해서 971번째 문자에서 끝난다는 것을 의미합니다.

2.  **추가 텍스트**: 당신이 대화 이외의 텍스트를 추가하면, Emacs 버퍼의 전체 문자 수가 변경됩니다. 이 때, 기존의 메시지 위치 정보는 더 이상 정확하지 않게 됩니다.

3.  **재계산**: 대화가 진행되거나 버퍼에 변경이 생길 때마다, Emacs는 `GPTEL_BOUNDS` 값을 재계산합니다. 이는 다음과 같은 방법으로 이루어집니다:
    -   각 메시지의 시작 위치는 그 메시지가 처음 기록된 위치에서부터 추가된 텍스트의 길이를 더한 값으로 업데이트됩니다.
    -   끝 위치는 시작 위치에 메시지의 길이를 더한 값으로 업데이트됩니다.

4.  **정확성**: 이 재계산 과정은 Emacs의 버퍼 변경 이벤트를 감지하고, 그에 따라 `GPTEL_BOUNDS` 값을 업데이트하는 방식으로 이루어집니다. 따라서, 버퍼에 추가된 텍스트의 길이와 위치에 따라 정확하게 수정됩니다.

#### @assistant  Yes, it is _possible_ to set different **temperature values** for LLM services in Emacs:

-   **Config**: You can define variables with unique names for each service, allowing different temperature settings:

<!--listend-->

```emacs-lisp
(defvar gptel-service-one-api-key "your-api-key"
  "API key for LLM service one.")

(defvar gptel-service-one-temperature 0.9
  "Temperature setting for LLM service one.")

(defvar gptel-service-two-api-key "your-api-key"
  "API key for LLM service two.")

(defvar gptel-service-two-temperature 0.5
  "Temperature setting for LLM service two.")
```

-   **Backend**: When calling `gptel-send`, you can provide _keyword arguments_ specifying the desired **backend** and **temperature** for each service:

<!--listend-->

```emacs-lisp
(gptel-send "Your text for service one" :api-key gptel-service-one-api-key :temperature gptel-service-one-temperature)

(gptel-send "Your text for service two" :api-key gptel-service-two-api-key :temperature gptel-service-two-temperature)
```

This setup allows you to maintain **per-backend settings** for temperature, ensuring customization for different services.
