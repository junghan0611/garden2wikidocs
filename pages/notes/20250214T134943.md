### z80dev/uv-mode

(“Z80dev/Uv-Mode” 2025)

-   z80
-   emacs integration for uv virtual environments

UV Mode automatically detects and activates virtual environments created by uv in your Python projects. When you open a Python file, UV Mode checks for a .venv directory in the project root and activates it if found. This enables seamless switching between different Python projects, each with its own isolated environment.

UV 모드는 Python 프로젝트에서 생성된 가상 환경을 자동으로 감지하고 활성화합니다. Python 파일을 열면 UV 모드는 프로젝트 루트에서 .venv 디렉터리를 확인하여 해당 디렉터리가 있으면 활성화합니다. 이렇게 하면 각각 독립된 환경을 가진 여러 Python 프로젝트 간에 원활하게 전환할 수 있습니다.

```elisp

(use-package uv-mode
  :hook (python-mode . uv-mode-auto-activate-hook))

```

#### Key features include:

-   Automatic virtual environment detection and activation
-   Mode line indicator showing the current project/environment
-   Integration with pythonic.el for enhanced Python development
-   Simple keybindings for manual control when needed

## venv 파일 인식 관련

@user 파이썬 패키지 매니저 uv에서 venv 파일을 인식한다고 하는데 이렇게 되면 기존 파이썬 패키지 툴과 충돌이 발생하지 않는가?

@assistant 파이썬 패키지 매니저인 `uv` 에서 `venv` 파일을 인식하는 것은 기존 파이썬 패키지 툴과 충돌을 일으킬 가능성이 낮습니다. `uv` 는 PEP 517, 518, 508, 660을 지원하여 표준화된 패키지 관리와 빌드 시스템을 제공하며, `venv` 와의 호환성을 유지합니다[3]. `uv` 는 `venv` 를 통해 생성된 가상 환경을 활용하여 개발 환경과 싱크를 맞출 수 있으며, 이는 기존의 `pip` 나 `venv` 와의 충돌보다는 보완적인 역할을 합니다[3]. 다만, 특정 폴더명이나 경로 문제로 인해 가상 환경이 제대로 인식되지 않는 경우가 있을 수 있으므로, 폴더명에 특수문자를 사용하지 않는 것이 좋습니다[2].

Citations: [1] <https://python-guide-kr.readthedocs.io/ko/latest/dev/virtualenvs.html> [2] <https://blog.jagaldol.com/development/python-venv-error/> [3] <https://www.0x00.kr/development/python/python-uv-simple-usage-and-example> [4] <https://www.daleseo.com/python-venv/> [5] <https://docs.astral.sh/uv/pip/environments/>

@user 혹시 venv 파일을 인식하는 다른 도구가 있는가?

@assistant **\*venv** 파일을 인식하는 도구로는 주로 **pipenv** 와 **virtualenv** 가 있습니다.

-   **pipenv**: 가상 환경을 만들고 패키지 의존성을 관리하는 도구로, **venv** 와 유사한 기능을 제공합니다[1][2].
-   **virtualenv**: Python 2.x에서 주로 사용되었으며, Python 3.x에서도 사용 가능합니다. **venv** 와 비슷하지만 더 많은 기능을 제공합니다[4].

이 외에도 **uv** 와 같은 새로운 도구들이 **venv** 와 호환성을 유지하며 등장하고 있습니다.

**Visual Studio Code** 와 같은 IDE도 **venv** 를 지원하여 개발 환경을 쉽게 설정할 수 있습니다[5].
