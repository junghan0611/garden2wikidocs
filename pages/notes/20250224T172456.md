### Write sequence notes or "folgezettel"

2\\. 시퀀스 노트 또는 "folgezettel" 작성

`denote-sequence` 패키지는 시퀀싱 체계를 사용하여 파일 이름을 지정하기 위한 `denote` 의 선택적 확장 기능을 제공합니다. 아이디어는 파일 간에 계층적 관계를 설정하여 하나의 내용이 논리적으로 다른 내용에 따르거나 보완하도록 하는 것입니다.

Denote는 `SIGNATURE` 이라는 선택적 파일 이름 구성 요소를 정의합니다(Denote 매뉴얼에서 파일 이름 지정 체계에 대해 읽어보세요). 이는 사용자가 원하는 텍스트로 채울 수 있는 자유 형식 필드입니다. 예를 들어 비디오를 `part1` 및 `part2` 로 분할하거나, `a` 및 `b` 와 같은 일종의 우선 순위를 설정하거나, 나머지 키워드와 눈에 띄는 특수 태그를 가질 수도 있습니다.

A more specialised use-case of the `SIGNATURE` is to define a hierarchical relationship between notes, such that the thoughts they expound on form sequences. For example, an article about the Labrador Retriever dog breed is a continuation of a thought process that extends something about dog breeds in general which, in turn, is a topic that belongs to the wider theme of dogs. A sequence, then, is a representation of such relationships. A note with a `SIGNATURE` of `1=1` (the `=` is the field separator of signatures, per the Denote file-naming scheme) is thus the first child of note `1` and the sibling of note `1=2`. In this regard, something unrelated to dogs will be its own parent, such as `2` , and so on.

`SIGNATURE` 의 더 전문화된 사용 사례는 노트 간의 계층적 관계를 정의하여 노트가 설명하는 생각이 시퀀스를 형성하도록 하는 것입니다. 예를 들어 래브라도 리트리버 견종에 대한 기사는 일반적으로 견종에 대한 무언가를 확장하는 사고 과정의 연속이며, 이는 차례로 개라는 더 넓은 주제에 속합니다. 따라서 시퀀스는 이러한 관계의 표현입니다. `1=1` 의 `SIGNATURE` ( `=` 은 Denote 파일 이름 지정 체계에 따른 서명 필드 구분 기호임)이 있는 노트는 노트 `1` 의 첫 번째 자식이자 노트 `1=2` 의 형제입니다. 이와 관련하여 개와 관련 없는 것은 `2` 와 같이 자체 부모가 됩니다.

All the relevant functions we provide take care to automatically use the right number for a given sequence ([Create parent, child, or sibling sequence notes](https://protesilaos.com/emacs/denote-sequence#h:create-parent-child-or-sibling-sequence-notes)). If, for example, we create a new child of parent `1=1`, we make sure that it is the largest number among any existing children, so if `1=1=1` already exists we use `1=1=2` , and the like.

우리가 제공하는 모든 관련 함수는 주어진 시퀀스에 대해 올바른 숫자를 자동으로 사용하도록 주의합니다(부모, 자식 또는 형제 시퀀스 노트 생성). 예를 들어 부모 `1=1` 의 새 자식을 생성하는 경우 기존 자식 중 가장 큰 숫자인지 확인합니다. 따라서 `1=1=1` 이 이미 존재하는 경우 `1=1=2` 를 사용하는 것과 같습니다.

The `denote-sequence.el` optional extension is not necessary for such a workflow. Users can always define whatever `SIGNATURE` they want manually. The purpose of this extension is to streamline this work.

선택적 `denote-sequence.el` 확장 기능은 이러한 워크플로에 필요하지 않습니다. 사용자는 항상 원하는 `SIGNATURE` 을 수동으로 정의할 수 있습니다. 이 확장 기능의 목적은 이 작업을 간소화하는 것입니다.

#### 2.1. Select a sequencing scheme for `denote-sequence-scheme`

2.1. `denote-sequence-scheme` 에 대한 시퀀싱 체계를 선택합니다.

사용자 옵션 `denote-sequence-scheme` 을 통해 사용자는 `1=1=2` 와 같은 `numeric` 체계 또는 동일한 시퀀스에 대해 `1a2` 인 `alphanumeric` 체계를 선택할 수 있습니다(하나의 시퀀싱 체계를 다른 체계로 변환)

##### Numeric sequencing scheme 숫자 시퀀싱 체계

A numeric sequence consists only of numbers. The level of depth is derived from the number of fields in the sequence, separated by the equals sign. Thus, the sequence `1=1=2` consists of three levels of depth. For deeper sequences, the numeric scheme will get longer, which some users may consider unwieldy. The upside, however, is that is easier to reason about larger numbers, such as `1=100=2=50` .

숫자 시퀀스는 숫자로만 구성됩니다. 깊이 수준은 등호로 구분된 시퀀스의 필드 수에서 파생됩니다. 따라서 `1=1=2` 시퀀스는 3단계 깊이로 구성됩니다. 더 깊은 시퀀스의 경우 숫자 체계가 더 길어지며, 일부 사용자는 이를 다루기 어렵다고 생각할 수 있습니다. 그러나 장점은 `1=100=2=50` 과 같은 큰 숫자에 대해 더 쉽게 추론할 수 있다는 것입니다.

##### Alphanumeric sequencing scheme 영숫자 시퀀싱 체계

An alphanumeric sequence combines numbers and letters. The level of depth is undestand by the alteration from numbers to letters and vice versa. As such, the sequence `1a2` has three levels of depth. This scheme is more compact, which users may like but can be harder to reason about large numbers, such as `1zzzv2zx` corresponding to the numeric `1=100=2=50` (this is because the number 26 is z, 27 is za, 52 is zz, and so on). In practice, large numbers may not be a problem, though this is something to keep in mind.

영숫자 시퀀스는 숫자와 문자를 결합합니다. 깊이 수준은 숫자에서 문자로, 또는 그 반대로의 변경에 따라 이해됩니다. 따라서 시퀀스 `1a2` 은 세 개의 깊이 수준을 가집니다. 이 체계는 더 간결하여 사용자가 선호할 수 있지만, `1zzzv2zx` 이 숫자 `1=100=2=50` 에 해당하는 것과 같이 큰 숫자를 추론하기가 더 어려울 수 있습니다 (이는 숫자 26이 z이고, 27은 za, 52는 zz 등이기 때문입니다). 실제로 큰 숫자는 문제가 되지 않을 수 있지만, 염두에 두어야 할 사항입니다.

##### 2.1.1. Convert from one sequencing scheme to another

2.1.1. 한 시퀀싱 체계에서 다른 체계로 변환

The decision on the desired `denote-sequence-scheme` wil affect new notes long-term ([Select a sequencing scheme for `denote-sequence-scheme`](https://protesilaos.com/emacs/denote-sequence#h:select-a-sequencing-scheme-for-denote-sequence-scheme)). It thus is important to think through your needs and proceed accordingly.

원하는 `denote-sequence-scheme` 에 대한 결정은 새 노트에 장기적으로 영향을 미칩니다 ( `denote-sequence-scheme` 에 대한 시퀀싱 체계를 선택하십시오). 따라서 사용자의 요구 사항을 신중하게 고려하고 그에 따라 진행하는 것이 중요합니다.

Still, one cannot be sure which scheme they prefer until they experiment with it. It then is inconvenient to manually revert to the alternative scheme. To this end, we provide the command `denote-sequence-convert` . It convers one or more files from their current scheme to its counterpart.

여전히 어떤 스킴을 선호하는지 실험해 보기 전에는 확신할 수 없습니다. 그런 다음에는 수동으로 다른 스킴으로 되돌리는 것이 불편합니다. 이를 위해 `denote-sequence-convert` 명령을 제공합니다. 현재 스킴에서 해당 스킴으로 하나 이상의 파일을 변환합니다.

When called from inside a Denote file, it converts that file. When called from a Dired buffer, it operates on the marked files. If no files are marked, it works with the Dired file at point.

Denote 파일 내에서 호출되면 해당 파일을 변환합니다. Dired 버퍼에서 호출되면 표시된 파일에 대해 작동합니다. 표시된 파일이 없으면 현재 포인트의 Dired 파일로 작업합니다.

Note that `denote-sequence-convert` DOES NOT REPARENT OR ANYHOW CHECK THE RESULTING SEQUENCES FOR DUPLICATES ([Re-parent a file to extend a given sequence](https://protesilaos.com/emacs/denote-sequence#h:re-parent-a-file-to-extend-a-given-sequence)).

`denote-sequence-convert` 은(는) 결과 시퀀스의 부모를 변경하거나 중복 항목을 확인하지 않습니다. (파일의 부모를 변경하여 주어진 시퀀스를 확장합니다.)

#### 2.2. Create parent, child, or sibling sequence notes

2.2. 부모, 자식 또는 형제 시퀀스 노트 생성

[ In the interest of simplicity, here we provide examples using the `numeric` value of `denote-sequence-scheme`, though the `alphanumeric` will work as well ([Select a sequencing scheme for `denote-sequence-scheme`](https://protesilaos.com/emacs/denote-sequence#h:select-a-sequencing-scheme-for-denote-sequence-scheme)). ]

(간단하게 하기 위해 여기서는 `denote-sequence-scheme` 의 `numeric` 값을 사용하는 예제를 제공하지만 `alphanumeric` 도 작동합니다. ( `denote-sequence-scheme` 의 시퀀싱 체계를 선택합니다.))

A new sequence note can be of the type `parent`, `child`, and `sibling` . For the convenience of the user, we provide commands to create such "sequence notes", link only between them (as opposed to a link to any other file with the Denote file-naming scheme (read the Denote manual about link-related commands)), and re-parent them on demand.

새로운 시퀀스 노트는 `parent` , `child` 및 `sibling` 유형일 수 있습니다. 사용자의 편의를 위해 이러한 "시퀀스 노트"를 생성하고, 이러한 노트 간에만 링크를 연결하고 (Denote 파일 이름 지정 체계를 사용하는 다른 파일로의 링크가 아닌 경우, 링크 관련 명령에 대한 Denote 매뉴얼 참조), 필요에 따라 상위 항목을 다시 지정하는 명령을 제공합니다.

##### Concretely, we provide the following commands:

구체적으로 다음과 같은 명령을 제공합니다.

`denote-sequence`

The most general way to create a new sequence note. It prompts for a type of sequence among `parent`, `child`, and `sibling` and the rest of the work accordingly. If the new sequence is not a parent, it thus prompts for an existing file to extend from. The rest of the interaction is that of all the usual Denote commands, such as to prompt for a title and keywords (read the Denote manual about the main points of entry).

새 시퀀스 노트를 만드는 가장 일반적인 방법입니다. `parent` , `child` , `sibling` 중에서 시퀀스 유형을 묻고 그에 따라 나머지 작업을 수행합니다. 새 시퀀스가 상위 시퀀스가 아닌 경우 확장할 기존 파일을 묻습니다. 나머지 상호 작용은 제목 및 키워드를 묻는 것과 같이 일반적인 Denote 명령과 같습니다(주요 진입점에 대한 Denote 설명서를 참조하십시오).

`denote-sequence-new-parent`

This is a convenience wrapper of `denote-sequence` which directly creates a parent sequence.

이것은 `denote-sequence` 의 편리한 래퍼로, 상위 시퀀스를 직접 만듭니다.

`denote-sequence-new-child`

This is a convenience wrapper of `denote-sequence` which directly creates a child of an existing sequence, prompting for it using minibuffer completion.

이것은 `denote-sequence` 의 편리한 래퍼로, 미니버퍼 완성을 사용하여 기존 시퀀스의 하위 시퀀스를 직접 만들고 묻습니다.

`denote-sequence-new-child-of-current`

This will create a new child of the current file's sequence. If the current file does not have such a sequence, then the command behaves the same as the aforementioned `denote-sequence-new-child` .

이는 현재 파일의 시퀀스에 새로운 자식을 만듭니다. 현재 파일에 이러한 시퀀스가 없는 경우, 이 명령은 앞서 언급한 `denote-sequence-new-child` 과 동일하게 작동합니다.

`denote-sequence-new-sibling`

This is a convenience wrapper of `denote-sequence` which directly creates a sibling of an existing sequence, prompting for it using minibuffer completion. 이것은 미니버퍼 완성을 사용하여 기존 시퀀스의 형제를 직접 만들도록 프롬프트하는 `denote-sequence` 의 편리한 래퍼입니다.

`denote-sequence-new-sibling-of-current`

This will create a new sibling of the current file's sequence. If the current file does not have such a sequence, then the command behaves the same as the aforementioned `denote-sequence-new-sibling` .

이는 현재 파일의 시퀀스와 같은 레벨의 새로운 파일을 생성합니다. 현재 파일에 그러한 시퀀스가 없는 경우, 이 명령어는 앞서 언급한 `denote-sequence-new-sibling` 과 동일하게 작동합니다.

#### 2.3. Find a relative of the current sequence

2.3. 현재 시퀀스의 관련 파일 찾기

While reading a file with a sequence, you may want to find what its relatives are about. To this end, the command `denote-sequence-find` prompts for a type among `parent`, `sibling`, `child` , and then asks to select a file among those matching the given type. It then visits the file.

시퀀스가 있는 파일을 읽는 동안 해당 파일의 관련 파일이 무엇인지 확인하고 싶을 수 있습니다. 이를 위해 `denote-sequence-find` 명령은 `parent` , `sibling` , `child` 중에서 유형을 묻고, 주어진 유형과 일치하는 파일 중에서 하나를 선택하라는 메시지를 표시합니다. 그런 다음 해당 파일을 방문합니다.

Instead of selecting a single file, the command `denote-sequence-find-dired` puts all the matching files in a bespoke Dired buffer ([Show all or some sequences in a Dired buffer](https://protesilaos.com/emacs/denote-sequence#h:show-all-or-some-sequences-in-a-dired-buffer)). 단일 파일을 선택하는 대신 `denote-sequence-find-dired` 명령은 일치하는 모든 파일을 맞춤형 Dired 버퍼에 넣습니다 (Dired 버퍼에서 모든 시퀀스 또는 일부 시퀀스 표시).

#### 2.4. Link only to sequences

2.4. 시퀀스로만 연결하기

The command `denote-sequence-link` is a variant of the standard `denote-link` command which limits the list of files only to those which contain a sequence (read the Denote manual about link-related commands). Consider it a convenience to link to sequence notes more quickly. It is by no means necessary though, as the regular linking commands will work as expected with any Denote file, including those which contain a sequence as their file name `SIGNATURE` ([Write sequence notes or "folgezettel"](https://protesilaos.com/emacs/denote-sequence#h:write-sequence-notes-or-folgezettel)).

명령어 `denote-sequence-link` 은 표준 `denote-link` 명령어의 변형이며, 파일 목록을 시퀀스를 포함하는 파일로만 제한합니다(링크 관련 명령어에 대한 Denote 매뉴얼을 참조하십시오). 시퀀스 노트에 더 빠르게 연결할 수 있는 편리한 기능이라고 생각하십시오. 그러나 일반 링크 명령은 파일 이름 `SIGNATURE` (시퀀스 노트 또는 "folgezettel" 작성)로 시퀀스를 포함하는 파일을 포함하여 모든 Denote 파일에서 예상대로 작동하므로 반드시 필요한 것은 아닙니다.

#### 2.5. Re-parent a file to extend a given sequence

2.5. 주어진 시퀀스를 확장하기 위해 파일의 부모를 다시 지정하기

The command `denote-sequence-reparent` can be used from inside a file or for the file-at-point in Dired to make that file a child of a given sequence. It does so by prompting for the target file using minibuffer completion. Niklas Luhmann would edit notes to form sequences of thoughts with branching paths, such as `1.1`, `1.1a`, `1.2`, `1.2a`, `1.2b`, etc. With the Denote file-naming scheme, we make the word separator in each file name component use the same character as the entire field, so words in a title have a dash between them and signatures have the equals sign (The file-naming scheme). Thus, our Luhmann-style signature will be slightly different in their looks: `1=1`, `1=1a`, `1=2`, `1=2a`, `1=2b`.

When using the `denote-sort-dired` command with default settings, our signatures will not sort in an intuitive way. This is because they combine numbers and letters, which require a different approach than what the default sorting function is using (Define a sorting function per component). In the following code block, we show a sorting algorithm that should do the right thing while dealing with Luhmann-style signatures.

```emacs-lisp
(defun my-denote--split-luhman-sig (signature)
  "Split numbers and letters in Luhmann-style SIGNATURE string."
  (replace-regexp-in-string
   "\\([a-zA-Z]+?\\)\\([0-9]\\)" "\\1=\\2"
   (replace-regexp-in-string
    "\\([0-9]+?\\)\\([a-zA-Z]\\)" "\\1=\\2"
    signature)))

(defun my-denote--pad-sig (signature)
  "Create a new signature with padded spaces for all components"
  (combine-and-quote-strings
   (mapcar
    (lambda (x)
      (string-pad x 5 32 t))
    (split-string (my-denote--split-luhman-sig signature) "=" t))
   "="))

(defun my-denote-sort-for-signatures (sig1 sig2)
  "Return non-nil if SIG1 is smaller that SIG2.
Perform the comparison with `string<'."
  (string< (my-denote--pad-sig sig1) (my-denote--pad-sig sig2)))

;; Change the sorting function only when we sort by signature.
(setq denote-sort-signature-comparison-function #'my-denote-sort-for-signatures)
```
