@assistant Doom Emacs의 **editor 모듈** 는 여러 가지 기능을 제공합니다. 주요 기능은 다음과 같습니다:

-   **evil 모듈**: Vim과 유사한 키바인딩을 제공하여 Vim 사용자에게 친숙한 환경을 제공합니다. evil 모듈은 다양한 Vim 플러그인의 기능을 포함하고 있습니다[4]. - **file-templates 모듈**: yasnippet을 사용하여 빈 파일에 템플릿을 자동으로 삽입할 수 있습니다. 템플릿은 특정 파일 형식이나 경로에 따라 자동으로 확장됩니다[2]. - **multiple-cursors 모듈**: 여러 커서를 사용하여 동시에 여러 위치에서 편집할 수 있는 기능을 제공합니다[4]. 이 외에도 Doom Emacs는 다양한 모듈을 통해 사용자에게 편리한 편집 환경을 제공합니다. - Citations: [1] <https://zzamboni.org/post/my-doom-emacs-configuration-with-commentary/> [2] <https://github.com/doomemacs/doomemacs/blob/master/modules/editor/file-templates/README.org> [3] <https://valerioviperino.me/doom-emacs-handbook/> [4] <https://github.com/doomemacs/doomemacs/blob/master/modules/editor/evil/README.org> [5] <https://aquabeam.me/tech/doom_emacs_intro/> user more in deltails about belows on the editor modules evil/ file-templates/ fold/ format/ god/ lispy/ multiple-cursors/ objed/ parinfer/ rotate-text/ snippets/ word-wrap/ assistant Doom Emacs의 **editor 모듈** 에 대해 자세히 설명드리겠습니다. 1. evil - **설명**: Vim과 유사한 키바인딩을 제공하는 모듈입니다. Evil은 Vim의 기능을 Emacs에서 사용할 수 있게 해줍니다[4]. - **기능**: Vim의 이동, 편집 명령어를 지원하며, Evil Collection을 기반으로 하여 다양한 Vim 플러그인의 기능을 포함하고 있습니다[4]. 2. file-templates - **설명**: 파일 템플릿을 자동으로 삽입해주는 모듈입니다. yasnippet을 사용하여 특정 파일 형식이나 경로에 맞는 템플릿을 생성합니다[2]. 3. fold - **설명**: 코드를折り畳む 기능을 제공하는 모듈입니다. 코드의 특정 부분을 숨기거나 보여주는 데 사용됩니다. 4. format - **설명**: 코드를 자동으로 정렬하고 포맷팅하는 모듈입니다. 다양한 언어에 맞는 포맷팅 규칙을 지원합니다. 5. god - **설명**: 현재 검색 결과에서는 god 모듈에 대한 정보가 없습니다. 6. lispy - **설명**: Lisp 코드를 작성할 때 유용한 모듈입니다. Vim과 유사한 키바인딩을 제공하며, Lisp 코드의 구조를 쉽게 관리할 수 있게 도와줍니다[5]. 7. multiple-cursors - **설명**: 여러 커서를 사용하여 동시에 여러 위치에서 편집할 수 있는 기능을 제공합니다[3]. 8. objed - **설명**: 현재 검색 결과에서는 objed 모듈에 대한 정보가 없습니다. 9. parinfer - **설명**: Lisp 코드의 들여쓰기를 자동으로 조정해주는 모듈입니다. 코드의 구조를 쉽게 관리할 수 있게 도와줍니다. 10. rotate-text - **설명**: 현재 검색 결과에서는 rotate-text 모듈에 대한 정보가 없습니다. 11. snippets - **설명**: 코드 스니펫을 쉽게 삽입할 수 있는 모듈입니다. yasnippet을 기반으로 하여 다양한 코드 템플릿을 제공합니다. 12. word-wrap - **설명**: 긴 줄의 텍스트를 자동으로 줄 바꿈하는 기능을 제공하는 모듈입니다. 읽기 편한 형태로 텍스트를 표시합니다. 이 외에도 다양한 모듈이 있으며, 각 모듈은 특정한 기능을 제공하여 사용자에게 편리한 편집 환경을 조성합니다. - Citations: [1] <https://discourse.doomemacs.org/t/updated-information-on-how-to-disable-evil-mode/2674> [2] <https://github.com/doomemacs/doomemacs/blob/master/modules/config/default/+evil-bindings.el> [3] <https://valerioviperino.me/doom-emacs-handbook/> [4] <https://docs.doomemacs.org/v21.12/modules/editor/evil/> [5] <https://org-roam.discourse.group/t/package-for-writing-elisp-code-as-doom-emacs-evil-user/1560> user evil/ file-templates/ fold/ [코드폴딩 vimish-fold ts-fold hideshow](https://notes.junghanacs.com/notes/20250323T073347/) 여기서 다룬다.
