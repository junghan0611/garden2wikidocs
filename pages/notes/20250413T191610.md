##### **Advent of Code(AoC) 소개**

매년 12월 1일부터 25일까지 진행되는 크리스마스 테마의 프로그래밍 대회로, **에릭 와슬(Eric Wastl)** 이 2015년부터 운영 중입니다[3]. 매일 자정(EST 기준)에 공개되는 문제는 **두 부분** 으로 구성되며, 각 부분을 해결하면 별을 획득합니다. 참가자는 **모든 프로그래밍 언어** 를 사용할 수 있으며, 입력 데이터는 개인별로 다릅니다[3][5].

##### **다른 코딩 훈련과의 차이점**

| 구분      | AoC                            | 일반 코딩 대회/훈련 |
|---------|--------------------------------|-------------|
| **주제**  | 크리스마스 테마의 스토리 라인  | 특정 주제 없음    |
| **문제 구조** | 25일간의 일일 퍼즐, 2단계 문제 | 단일 문제 또는 특정 주제 집중 |
| **언어 선택** | **새로운 언어 학습** 에 초점[1] | 특정 언어 제한 없음 |
| **커뮤니티** | Reddit, GitHub 등 활발한 활동[5] | 제한적 커뮤니티   |
| **평가 기준** | **속도 경쟁** (1~100점) + **학습**[1] | 순수 점수/시간 경쟁 |

##### **문제 유형 및 난이도**

###### **주요 유형**

1.  **알고리즘 구현**: 정렬, 그래프 탐색, 동적 계획법 등[2][4].
2.  **데이터 구조 활용**: 리스트, 딕셔너리, 트리 등 효율적 처리[5].
3.  **입력 파싱**: ASCII 아트, 복잡한 텍스트 형식 처리[5].
4.  **시뮬레이션**: 규칙 기반 시스템 모델링[3].

##### **난이도**

-   **초보자**: 기본적인 입력 처리, 단순 계산 (예: 2024년 Day 1의 정렬 및 차이 계산)[2][4].
-   **중급자**: 최적화 기법, 재귀/반복 구조 활용.
-   **숙련자**: 대용량 데이터 처리, 복잡한 알고리즘 (예: 2023년 Day 7의 파일 시스템 시뮬레이션).

---

##### **실무 연관성**

1.  **문제 해결 접근법**: **"단계적 분해"** 를 요구하며, 실무에서 복잡한 시스템을 분석하는 데 유사한 사고 방식을 훈련합니다[5].
2.  **효율성**: 대용량 데이터 처리 시 **시간/공간 복잡도** 를 고려해야 하며, 이는 실무에서 성능 최적화와 직접 연결됩니다[5].
3.  **다양한 언어 경험**: **새로운 언어** 를 사용해 문제를 해결함으로써 **패러다임 확장** (예: 함수형 언어, 동시성 처리 언어)을 경험할 수 있습니다[1].

---

##### **참여 팁**

-   **언어 선택**: **새로운 언어** 를 선택해 학습에 집중 (예: OCaml, Julia, ReScript)[1].
-   **커뮤니티 활용**: Reddit 의 r/adventofcode 에서 **다양한 접근법** 을 공유하고 학습[5].
-   **테스트 데이터**: **샘플 입력** 으로 로직 검증 후 **실제 데이터** 제출[1].
