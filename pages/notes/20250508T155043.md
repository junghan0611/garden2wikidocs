##### 설치 및 실행 과정 (Step-by-Step Guide)

Windows 환경에서 GitHub CLI(깃허브 명령줄 인터페이스)를 설치하고, 주어진 명령어를 실행하여 특정 레포지토리를 클론하는 과정을 아래에 단계별로 정리하였습니다. 초보자도 쉽게 따라 할 수 있도록 상세히 설명합니다.

1.  **GitHub CLI 설치하기**
    -   **공식 웹사이트 방문**: 브라우저에서 GitHub CLI 공식 다운로드 페이지(`https://cli.github.com/`)로 이동합니다.
    -   **Windows용 설치 파일 다운로드**: 페이지에서 Windows용 설치 파일(`.msi` 파일)을 다운로드합니다.
    -   **설치 실행**: 다운로드한 파일을 실행하여 설치 마법사를 따라 진행합니다. 기본 설정으로 설치하면 됩니다.
    -   **설치 확인**: 설치가 완료되면 명령 프롬프트(CMD) 또는 PowerShell을 열고 아래 명령어를 입력하여 설치 여부를 확인합니다.
        ```nil
        gh --version
        ```
        버전 정보가 출력되면 설치가 정상적으로 완료된 것입니다.

2.  **GitHub CLI 인증 설정**
    -   GitHub CLI를 사용하기 위해서는 GitHub 계정과 연동해야 합니다. 아래 명령어를 실행하여 인증을 진행합니다.
        ```nil
        gh auth login
        ```
    -   명령어를 실행하면 브라우저가 열리고 GitHub 로그인 페이지로 이동합니다. 로그인 후 인증 코드를 발급받아 터미널에 입력합니다.
    -   인증이 완료되면 GitHub CLI를 사용할 준비가 됩니다.

3.  **레포지토리 클론하기**
    -   원하는 디렉토리로 이동합니다. 예를 들어, `C:\Projects` 폴더에서 작업하고 싶다면 아래 명령어를 사용합니다.
        ```nil
        cd C:\Projects
        ```
    -   이제 주어진 명령어를 실행하여 `needleworm/bhban_rpa` 레포지토리를 클론합니다.
        ```nil
        gh repo clone needleworm/bhban_rpa
        ```
    -   클론이 완료되면 해당 디렉토리에 `bhban_rpa` 폴더가 생성됩니다. 폴더 안으로 이동하여 내용을 확인할 수 있습니다.
        ```nil
        cd bhban_rpa
        dir
        ```

4.  **문제 해결 팁**
    -   `gh` 명령어가 인식되지 않는 경우: 환경 변수(Path)에 GitHub CLI 경로가 추가되지 않았을 가능성이 있습니다. 설치 경로(기본적으로 `C:\Program Files\GitHub CLI`)를 환경 변수에 추가하세요.
    -   인증 오류: GitHub 계정의 2FA(2단계 인증)가 활성화되어 있다면, 개인 액세스 토큰(Personal Access Token)을 생성하여 인증에 사용해야 할 수 있습니다.
