### GPTStack ¤ufw 포트 요구사항

-   [GPUStack AI 클러스터 GPU 관리자 오픈소스](https://wikidocs.net/381760)

이 문서 조건에 맞도록 우분투에서 포트를 오픈하는 쉘 명령어를 정리 <https://docs.gpustack.ai/latest/installation/installation-requirements/#port-requirements>

Summarize the shell commands to open required ports for GPUStack installation on Ubuntu according to the official documentation.

아래는 GPUStack 공식 문서의 포트 요구사항(Port Requirements)[^fn:1]에 따라 **우분투(Ubuntu)** 에서 필요한 포트를 오픈(open)하는 대표적인 쉘 명령어(ufw, Uncomplicated Firewall 기준)입니다.

> **필수 포트(Ports) 및 용도(Description)는 공식 문서 기준이며, Ray(분산 처리 프레임워크) 활성화 여부에 따라 추가 포트가 필요합니다.**

##### 1. 서버(Server)용 포트 오픈

-   **UI 및 API 기본 포트**
    -   `sudo ufw allow 80/tcp` # HTTP (GPUStack UI 및 API)
    -   `sudo ufw allow 443/tcp` # HTTPS (TLS 활성화 시)
-   **Ray 활성화 시 추가 포트**
    -   `sudo ufw allow 8265/tcp` # Ray dashboard
    -   `sudo ufw allow 40096/tcp` # Ray GCS server
    -   `sudo ufw allow 40097/tcp` # Ray Client Server
    -   `sudo ufw allow 40098/tcp` # Ray node manager
    -   `sudo ufw allow 40099/tcp` # Ray object manager

##### 2. 워커(Worker)용 포트 오픈

-   **기본 포트**
    -   `sudo ufw allow 10150/tcp` # GPUStack worker 기본 포트
    -   `sudo ufw allow 10151/tcp` # 메트릭(metrics) 노출 포트
-   **추론 서비스용 포트 범위**
    -   `sudo ufw allow 40000:40063/tcp` # inference services
-   **llama-box RPC 서버용 포트 범위**
    -   `sudo ufw allow 40064:40095/tcp` # llama-box RPC servers
-   **Ray 활성화 시 추가 포트**
    -   `sudo ufw allow 40098/tcp` # Ray node manager
    -   `sudo ufw allow 40099/tcp` # Ray object manager
    -   `sudo ufw allow 40100:40131/tcp` # Ray worker processes

**참고**

-   위 명령어는 `ufw` 가 활성화(Enable)되어 있다고 가정합니다.
-   방화벽 정책에 따라 추가 설정이 필요할 수 있습니다.
-   실제 운영 환경에서는 최소 권한 원칙(Principle of Least Privilege)을 적용해 꼭 필요한 포트만 오픈하는 것이 바람직합니다.
