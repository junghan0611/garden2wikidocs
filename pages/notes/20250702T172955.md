#### NEXTU 8x10G SFP L3 스위치 시리얼 연결 방법 (Ubuntu Linux)

_연결 절차:_

1.  **필요한 도구 확인**
    -   USB-to-Serial 케이블 (일반적으로 FTDI 칩셋 사용)
    -   RJ45 콘솔 케이블 (스위치 제공 케이블 사용)

2.  **시리얼 포트 확인**
    ```bash
    ls /dev/ttyUSB*  # USB-to-Serial 연결 시
    ls /dev/ttyS*    # 내장 시리얼 포트 사용 시
    ```
    
    -   dmesg 출력에서 =ttyS1=(I/O 0x2f8)이 기본 시리얼 포트로 확인됨

3.  **minicom 설치 및 설정**
    ```bash
    sudo apt install minicom
    sudo minicom -s
    ```
    
    -   설정 메뉴에서:
        -   Serial Device: `/dev/ttyS1` (또는 해당 포트)
        -   Bps/Par/Bits: 115200 8N1
        -   Hardware Flow Control: No

4.  **물리적 연결**
    -   스위치의 Console 포트 ↔ Linux 서버의 시리얼 포트 연결
    -   전원 케이블도 함께 연결

5.  **접속 실행**
    ```bash
    sudo minicom -D /dev/ttyS1
    ```
    
    -   Enter 키를 여러 번 눌러 로그인 프롬프트 유도

_주의사항:_

-   권한 문제 시 `sudo usermod -aG dialout $USER` 실행 후 재로그인
-   CTS/RTS 플로우 컨트롤 비활성화 필수
-   케이블 불량 확인을 위해 `screen` 이나 `cu` 로도 테스트 가능

_추가 명령어:_

```bash
# 포트 설정 확인
stty -F /dev/ttyS1 -a
# raw 데이터 보기
cat /dev/ttyS1
```

_연결 후 작업:_

-   스위치 IP 설정 확인/변경
-   10GbE 포트 활성화 상태 점검
-   Jumbo Frame 설정 확인 (일반적으로 9000 MTU)

@assistant **\*ttyS1** 이 스위치 연결에 더 적합합니다.

**이유:**

1.  **속도 (Baud Rate)**
    -   ttyS1: 115200 baud → 네트워크 장비 표준 속도
    -   ttyS4: 9600 baud → 너무 느림 (구형 장비용)

2.  **플로우 컨트롤**
    -   ttyS1: `crtscts` (하드웨어 플로우 컨트롤) → 네트워크 장비에 적합
    -   ttyS4: `-crtscts` (플로우 컨트롤 비활성화)

3.  **일반적인 네트워크 장비 설정**
    -   대부분의 현대 스위치는 115200 8N1 설정 사용
    -   NEXTU 같은 관리형 스위치는 115200이 표준

**연결 시도:**

```bash
sudo minicom -D /dev/ttyS1
```

**minicom 설정:**

-   Serial Device: `/dev/ttyS1`
-   Bps/Par/Bits: 115200 8N1
-   Hardware Flow Control: Yes (crtscts 활성화됨)

**참고:**

-   ttyS4는 ARM 기반 내장 UART로 보이며 (dw-apb-uart), 일반적으로 시스템 디버깅용
-   ttyS1은 표준 16550A 시리얼 포트로 외부 장비 연결에 적합

**따라서 ttyS1(/dev/ttyS1)로 스위치에 연결하세요.**
