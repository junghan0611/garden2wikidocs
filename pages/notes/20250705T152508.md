## 무선랜 설정 방법

### 3. 무선 네트워크 설정

#### 3-1. 네트워크 인터페이스 확인

```sh
ip a
```

-   `wlx...` 또는 `wlan0` 등으로 무선랜 인터페이스가 보이면 정상 인식[4].

#### 3-2. Netplan(우분투 18.04 이상) 설정 예시

```yaml
# /etc/netplan/01-netcfg.yaml (파일명은 다를 수 있음)
network:
  version: 2
  wifis:
    wlx90de806d3a87:  # 실제 인터페이스명으로 변경
      dhcp4: true
      access-points:
        "SSID명":
          password: "비밀번호"
```

-   저장 후 적용:

<!--listend-->

```sh
sudo netplan apply
```

-   DHCP가 아닌 고정 IP를 원할 경우 `dhcp4: false` 및 `addresses`, `gateway4`, `nameservers` 항목 추가[5].

### 4. br_netfilter 관련 메시지 해결

-   커널 메시지에서 `br_netfilter` 관련 경고가 보이면, 아래 명령으로 모듈을 로드합니다.

<!--listend-->

```sh
sudo modprobe br_netfilter
echo "br_netfilter" | sudo tee /etc/modules-load.d/br_netfilter.conf
```

-   부팅 시 자동 로드 설정[6][7][8].

### 5. 문제 해결 팁

-   드라이버 설치 후에도 네트워크가 안 잡히면, `dmesg`, `lsusb`, `ip a` 등으로 상태 확인.
-   `rtl8188fu-dkms` 패키지 설치 후에도 인식이 안 되면, 커널 버전 호환성 확인 및 수동 빌드 시도.
-   Netplan 설정 후 `sudo netplan apply` 로 적용, 필요시 재부팅.

### 6. 참고 명령어 요약

| 목적            | 명령어 예시                            |
|---------------|-----------------------------------|
| 드라이버 설치(PPA) | `sudo apt install rtl8188fu-dkms`      |
| 드라이버 설치(소스) | `git clone ... && sudo dkms install .` |
| 무선랜 확인     | `ip a`, `iwconfig`, `lsusb`            |
| netplan 적용    | `sudo netplan apply`                   |

## Netplan 기반 우분투 서버 Wi-Fi 설정: 불필요한 패키지 정리

### 1. 패키지별 역할 요약

| 패키지명        | 주요 역할 및 필요성 (Netplan 기준) |
|-------------|--------------------------|
| wpasupplicant   | WPA/WPA2 무선 인증 데몬. **필수** |
| wireless-tools  | 구식 무선랜 관리 도구(iwconfig 등). **불필요** |
| iw              | 무선랜 상태 확인/설정(저수준). **선택적** |
| network-manager | 데스크탑용 네트워크 관리 데몬. **불필요** |

-   **Netplan** 은 네트워크 설정을 YAML 파일로 관리하며, 실제 무선 연결은 내부적으로 `systemd-networkd` 또는 `NetworkManager` 백엔드를 사용합니다.
-   **서버 환경** 에서 Netplan을 `networkd` 백엔드로 사용한다면, `network-manager` 는 필요하지 않습니다.
-   **wpasupplicant** 는 무선랜 인증에 반드시 필요합니다.
-   **iw** 는 무선랜 상태 확인용으로 있으면 편리하지만, 필수는 아닙니다.
-   **wireless-tools** 는 구식 도구로, 최신 환경에서는 거의 사용하지 않습니다.

### 2. 삭제해도 되는 패키지

-   **network-manager**: Netplan이 `networkd` 백엔드로 동작한다면 불필요합니다.
-   **wireless-tools**: 최신 환경에서는 필요 없습니다.
-   **iw**: 무선랜 상태 확인용이지만, 꼭 필요하지는 않습니다.

### 3. 남겨야 할 패키지

-   **wpasupplicant**: 무선랜 인증에 반드시 필요합니다.

### 4. 정리: 삭제/유지 권장 표

| 패키지명        | 삭제 권장 | 유지 권장 | 비고            |
|-------------|-------|-------|---------------|
| wpasupplicant   |       | ●     | 무선랜 인증 필수 |
| wireless-tools  | ●     |       | 구식, Netplan과 무관 |
| iw              | ●     |       | 상태 확인용, 선택적 |
| network-manager | ●     |       | 데스크탑/GUI 환경용 |

### 5. 삭제 명령 예시

```sh
sudo apt purge network-manager wireless-tools iw
sudo apt autoremove
```

-   `wpasupplicant` 만 남기고 나머지는 삭제해도 Netplan+systemd-networkd 환경에서 무선랜 연결에 문제 없습니다.
